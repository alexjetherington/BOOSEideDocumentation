Table of Contents

Namespace BOOSEide

Classes

AppBoolean

Represents a boolean variable evaluation command within a stored program. Mirrors the functionality of BOOSE.Boolean using AppEvaluation and AppStoredProgram.

AppCanvas

Application-specific canvas implementing IAppCanvas. Provides drawing functionality for shapes, lines, and text using a bitmap-backed graphics surface.

AppCanvasCommand

Represents a command that interacts with a canvas. Replaces the original BOOSE.CanvasCommand in the App* hierarchy.

AppCast

Application-specific cast command. Casts a real (AppReal) value to an integer (AppInt) variable.

AppCircle

Represents a circle drawing command that draws a circle centred on the current pen position of the canvas.

AppCommand

Mirrors the functionality of BOOSE.Command but adapted for AppStoredProgram and IAppCommand. Provides base functionality for all App* commands including parameter parsing, compilation, and execution.

AppCommandFactory

Factory for creating application commands (App* classes). Returns instances of AppCommand-derived classes such as AppMoveTo, AppDrawTo, AppCircle, etc.

AppCommandOneParameter

Base class for application-specific commands that take a single parameter. Works with AppCanvasCommand, AppStoredProgram, and AppEvaluation.

AppCommandThreeParameter

Base class for commands that require exactly three parameters. Inherits from AppCommandTwoParameter and provides evaluation and assignment for the third parameter.

AppCommandTwoParameter

Base class for commands that require exactly two parameters. Inherits from AppCommandOneParameter and provides evaluation and assignment for the second parameter.

AppCompoundCommand

Represents a compound command that contains a body, such as IF, WHILE, or FOR blocks. Mirrors the functionality of BOOSE.CompoundCommand using AppConditionalCommand and AppBoolean. Handles compile-time metadata, including parameter checking and linkage to corresponding end commands, but does not execute the body itself.

AppConditionalCommand

Represents metadata for conditional commands such as IF, WHILE, or FOR. Mirrors BOOSE.ConditionalCommand using AppBoolean and AppStoredProgram. Stores line numbers, condition evaluation, and type information. Execution of the block body is handled by AppCompoundCommand.

AppDrawTo

Draws a line from the current pen position to the specified (x, y) coordinates on the canvas. Inherits from AppCommandTwoParameter and works with IAppCanvas.

AppElse

Represents the ELSE part of an IF statement. Works in conjunction with AppIf and AppCompoundCommand. Responsible for skipping either the IF or ELSE block at runtime depending on the IF condition.

AppEnd

Represents the "END" of a compound command such as IF, WHILE, FOR, or method blocks. Responsible for handling loop continuation for AppWhile and AppFor commands, and marking the end of IF/ELSE blocks.

AppEvaluation

Represents a variable evaluation command (int, real, boolean) within a stored program. Handles variable declarations, uninitialized variables, and assignments. Logic mirrors the base BOOSE Evaluation class to ensure correct nested if handling.

AppFor

Represents a FOR loop command in a stored program. Handles loop control variable declaration, evaluation of "from", "to", and "step" expressions, and provides methods for loop continuation and incrementing. Execution of the loop body is handled by AppEnd and the BOOSE program engine.

AppIf

Represents an IF conditional command in the BOOSE language. Inherits from AppCompoundCommand to allow a block of commands to be conditionally executed.

AppInt

Application-specific integer evaluation class. Extends AppEvaluation to support integer variables and integer expression evaluation.

AppMoveto

Command that moves the pen to a specified x,y on the canvas without drawing. Inherits from AppCommandTwoParameter.

AppParser

Parses BOOSE script lines into IAppCommand instances. Properly separates left-hand side (variable) and right-hand side (expression) to avoid evaluating '=' prematurely.

AppPenColour

Sets the pen color on the canvas using three integer RGB components. Inherits from AppCommandThreeParameter to leverage automatic parameter handling.

AppReal

Represents a real number (double) variable in BOOSEide. Inherits from AppEvaluation and stores values as double.

AppRect

Draws a rectangle on a canvas using two integer parameters: width and height. Inherits from AppCommandTwoParameter to enforce exactly two parameters.

AppStoredProgram

Represents a stored BOOSE program containing AppCommand objects, variables, and method metadata. Supports execution, expression evaluation, and conditional block management.

AppWhile

Represents a WHILE loop command in the BOOSE language. Inherits from AppCompoundCommand to allow a block of commands to be repeatedly executed while a condition remains true.

AppWrite

Custom write command that outputs text to the canvas instead of the console. Inherits from AppCommandOneParameter to accept a single parameter.

GUI

Main GUI form for the BOOSE IDE application. Integrates the AppCanvas, IAppCommandFactory, AppStoredProgram, and IAppParser.

Interfaces

IAppCanvas

Application-specific canvas interface for use with AppCanvasCommand commands. Mirrors a standard ICanvas but is fully compatible with the App* command classes. Provides drawing operations, shape creation, text output, and canvas state management.

IAppCommand

Represents a generic command in a BOOSE stored program. All App* commands implement this interface.

IAppCommandFactory

Factory interface for creating application-specific commands. All App* commands should be created through a class implementing this interface.

IAppEvaluation

Represents a variable evaluation command within an AppStoredProgram. Extends IAppCommand to provide variable-specific properties and expression handling. Used by AppInt, AppReal, AppBoolean, and other variable commands.

IAppParser

Interface for a parser that converts BOOSE script lines or full programs into IAppCommand instances compatible with AppStoredProgram.

IAppStoredProgram

Interface representing a stored program that manages commands, variables, and program execution using App* classes.
