BOOSEide Documentation

Welcome to the BOOSEide IDE documentation. This includes both the user guides and the full API reference for developers.

Getting Started

  1. Download and install BOOSEide.
  2. Open the IDE.
  3. Use the input box to run BOOSE commands such as moveto, circle, rect, and others.
  4. The canvas will update as commands are executed.

Commands Overview

The following are some of the key commands available in BOOSEide:

  • moveto x,y – Moves the pen to a specific coordinate.
  • drawto x,y – Draws a line to the specified coordinate.
  • circle radius – Draws a circle at the current pen position.
  • rect width,height – Draws a rectangle from the current pen position.
  • pen r,g,b – Changes the pen color.
  • write "text" – Writes text at the current pen position.
  • int var = expression – Declares an integer variable.
  • real var = expression – Declares a real (double) variable.
  • boolean var = expression – Declares a boolean variable.
  • for var = from to to [step step] – Executes a loop.
  • while condition – Executes a loop while a condition is true.
  • if condition – Conditional execution.
  • end – Marks the end of a block (IF, FOR, WHILE, method).

For full details, see the API Reference below.

Interfaces

BOOSEide defines several core interfaces that standardize behavior for commands, evaluations, the canvas, and the parser:

  • IAppCanvas – Interface for the canvas used by drawing commands.
  • IAppCommand – Interface for all commands in the stored program.
  • IAppEvaluation – Interface for variables and evaluable expressions.
  • IAppParser – Interface for parsing lines and programs into commands.
  • IAppStoredProgram – Interface representing a stored program with variables and commands.
  • IAppCommandFactory – Interface for creating commands dynamically at runtime.

These interfaces allow you to extend BOOSEide with custom commands, parsers, or canvas implementations while remaining compatible with the existing architecture.

Testing / Unit Tests

BOOSEide includes automated tests to ensure that all commands, variables, and control structures work correctly. Key test classes include:

  • AppCanvasTests – Tests for drawing shapes, pen movement, and colors.
  • AppElseTests – Tests for conditional ELSE handling.
  • AppForTests – Tests for FOR loops and loop variable evaluation.
  • AppIfTests – Tests for IF conditional execution.
  • AppIntTests – Tests for integer variable declaration, assignment, and evaluation.
  • AppRealTests – Tests for real (double) variable declaration, assignment, and evaluation.
  • AppWhileTests – Tests for WHILE loop execution and termination.
  • TestCommandFactory – Tests for the command factory creating correct App* instances.

Running these tests ensures that all core functionalities behave as expected and helps prevent regressions during development.
